# amdyes

[English](README.md) | **中文**

一个简洁、零依赖部署的 **AMD GPU 实时监控工具**，相当于 AMD 平台上的 `nvidia-smi` + `nvtop`。它在终端里以全屏 curses 界面持续刷新，同时展示**每张 GPU 的状态**和**占用 GPU 的进程列表**。

整个工具只有一个 Python 脚本，核心逻辑代码量极小——它直接复用系统自带的 `rocm-smi` 库，无需重新实现底层逻辑。

---

## ✨ 特性

- **单文件、易部署**：只有一个 `amdyes.py`，复用系统 `rocm-smi`，不需要安装额外的大型依赖。
- **GPU 概览面板**：型号、温度、性能等级（Perf）、功耗 / 上限、显存使用、显存占用率、GPU 利用率。
- **进程视图**：列出占用 GPU 的进程（USER / PID / GPU / 显存占用 / CPU 占用 / 完整命令行），并按用户名排序。
- **彩色利用率提示**：利用率 < 30% 绿色，30%–70% 黄色，≥ 70% 红色，表头与边框使用青色，一眼看出谁在满载。
- **每秒自动刷新**：采用差异刷新（先采集数据再 `erase`），尽量减少闪烁。
- **设备名兜底**：当系统 `amdgpu.ids` 过旧（常见于 docker 环境）导致取不到型号名时，会自动从上游下载最新数据库；若无法联网，则回退到脚本内嵌的完整副本。
- **界面干净**：在文件描述符层面屏蔽 `rocm-smi`/`libdrm` 的杂散错误输出，避免污染全屏界面。

---

## 📋 环境要求

- **操作系统**：Linux（依赖 `/sys/bus/pci`、`/dev/null` 等，需要已加载 `amdgpu` 内核驱动）
- **ROCm**：已安装并可使用 `rocm-smi`（脚本会通过 `which rocm-smi` 自动定位其安装路径）
- **Python**：Python 3，标准库 `curses`（Linux 自带）
- **psutil**：用于读取进程的用户名 / CPU 占用 / 命令行

安装 psutil：

```bash
pip3 install psutil
```

---

## 🚀 安装与使用

### 方式一：直接运行

```bash
python3 amdyes.py
```

### 方式二：安装为全局命令

```bash
git clone https://github.com/limitmhw/amdyes
chmod 777 ./amdyes/amdyes.py
sudo ln -s $(realpath ./amdyes/amdyes.py) /usr/bin/amdyes
amdyes
```

之后在任意目录直接输入 `amdyes` 即可启动。

### 退出

在界面中按 **`q`** 或 **`Q`** 退出。

## 📊 字段说明

### GPU 概览面板

| 列 | 含义 |
| --- | --- |
| `ID` | GPU 设备索引 |
| `DeviceName` | GPU 型号名称（取不到时回退到 `amdgpu.ids` 数据库） |
| `Temp` | 当前温度 |
| `Perf` | 性能等级（如 `AUTO`） |
| `Pwr:Usage/Cap` | 当前功耗 / 功耗上限 |
| `Memory-Usage` | 已用显存 / 总显存 + 显存忙碌率 |
| `GPU-Util` | GPU 利用率（带颜色：绿 / 黄 / 红） |

### 进程列表

| 列 | 含义 |
| --- | --- |
| `USER` | 进程所属用户 |
| `PID` | 进程 ID |
| `GPU` | 进程占用的 GPU 数量 / 索引 |
| `GPU MEMORY` | 进程占用的显存 |
| `CPU USAGE` | 进程的 CPU 占用率 |
| `COMMAND` | 进程完整命令行（最长截断至 100 字符） |

### 利用率配色

| 利用率 | 颜色 |
| --- | --- |
| `< 30%` | 🟢 绿色 |
| `30% – 70%` | 🟡 黄色 |
| `≥ 70%` | 🔴 红色 |

## ⚙️ 工作原理

`amdyes` 不重复造轮子，而是**动态加载系统里的 `rocm-smi` Python 模块**并复用其内部函数：

1. 通过 `which rocm-smi` 找到 `rocm-smi` 的真实路径，并把它作为模块导入。
2. 调用底层 `rsmi_*` 接口采集 GPU 状态与计算进程信息（`showPids`）。
3. 用 `psutil` 补充每个进程的用户名、CPU 占用与命令行。
4. 用 `curses` 渲染全屏界面，每秒刷新一次。

**设备名兜底**：当 `libdrm`/`rocm-smi` 因系统 `amdgpu.ids` 过旧而返回不到型号名时，脚本会按 PCI device id + revision 在数据库中查找。数据库加载策略为：优先临时下载 [上游最新版](https://gitlab.freedesktop.org/mesa/drm/-/raw/main/data/amdgpu.ids)，下载失败时回退到脚本末尾内嵌的完整副本，确保在无外网的 docker 环境中也能显示正确的 GPU 名称。

## 🛠️ 常见问题

**`rocm-smi command not find`**
脚本找不到 `rocm-smi`。请确认已正确安装 ROCm，且 `rocm-smi` 在 `PATH` 中（`which rocm-smi` 能输出路径）。

**进程列表为空**
说明当前没有进程在 KFD（Compute）上下文中占用 GPU，或当前用户权限不足。读取部分信息可能需要 `root` 权限，可尝试用 `sudo` 运行。

**型号显示为 `N/A`**
通常是系统 `amdgpu.ids` 过旧。脚本会自动尝试联网更新或使用内嵌数据库兜底；若仍为 `N/A`，可能是该型号尚未被数据库收录。

**界面显示错乱**
请适当增大终端窗口尺寸；过窄的终端会截断每行内容。
